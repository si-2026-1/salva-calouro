# Requisitos Não Funcionais

**Descrição:** Os requisitos não funcionais especificam os critérios de qualidade, desempenho, segurança, usabilidade e as restrições técnicas sob as quais o sistema **Salva Calouro** deve operar, garantindo uma experiência estável, ágil e segura para toda a comunidade acadêmica.

---

## Lista de Requisitos Não Funcionais (RNF)

* **RNF01 (Usabilidade e Responsividade)** – O sistema deve ser construído como uma aplicação web com design responsivo (Mobile-first), priorizando o uso otimizado em smartphones durante a locomoção física pelo campus.
* **RNF02 (Personalização Institucional)** – O sistema deve permitir a adaptação automática de conteúdo, fluxos curriculares e mapas de acordo com o curso selecionado pelo usuário.
* **RNF03 (Desempenho)** – O carregamento do mapa interativo, a plotagem de rotas internas e a renderização da interface gráfica do fluxo devem responder às ações do usuário em até 3 segundos.
* **RNF04 (Disponibilidade)** – O sistema deve estar disponível 24 horas por dia, garantindo estabilidade contínua principalmente nos períodos de pico de matrículas acadêmicas.
* **RNF05 (Segurança)** – O sistema deve garantir o armazenamento seguro das credenciais de acesso e a integridade de todas as informações do histórico escolar inseridas pelos estudantes.
* **RNF06 (Confiabilidade do Armazenamento)** – O sistema deve armazenar os dados de forma estruturada e confiável para permitir consultas rápidas à ementa das disciplinas e dados geoespaciais sem perda de informação.
* **RNF07 (Escalabilidade/Concorrência)** – A infraestrutura do sistema deve suportar acessos simultâneos de múltiplos usuários da comunidade universitária sem apresentar degradação nos tempos de resposta da aplicação.
* **RNF08 (Acessibilidade)** – A interface do usuário deve seguir padrões de acessibilidade web para garantir que calouros com diferentes necessidades consigam navegar pelo campus e planejar sua grade sem barreiras digitais.
