    Requisitos Funcionais

    Descrição: Os requisitos funcionais definem as ações, os comportamentos e as funcionalidades específicas que a plataforma Salva Calouro deve possuir para que os estudantes (calouros e veteranos) consigam realizar suas tarefas e interagir dentro do sistema.

    Lista de Requisitos Funcionais (RF)

    Cadastro e Gestão de Perfil

    • RF01 – O sistema deve permitir o cadastro de usuários.
    • RF02 – O sistema deve permitir login e autenticação segura de usuários.
    • RF03 – O sistema deve diferenciar os perfis de acesso e as permissões entre "Calouros" e "Veteranos".
    • RF04 – O sistema deve permitir que o usuário edite as informações do seu perfil.

    Mapa Interativo e Rotas

    • RF05 – O sistema deve exibir um mapa interativo com o espaço geográfico do campus.
    • RF06 – O sistema deve permitir a busca por localização de salas de aula, laboratórios, blocos, prédios e serviços (ex: Restaurante Universitário, secretarias).
    • RF07 – O sistema deve traçar e exibir a rota ideal de um ponto a outro dentro da universidade.
    • RF08 – O sistema deve permitir clicar em pontos de interesse no mapa para ver mais detalhes.

    Controle de Fluxo Curricular

    • RF09 – O sistema deve apresentar visualmente a grade curricular do curso do usuário.
    • RF10 – O sistema deve permitir que o aluno marque as disciplinas que já foram cursadas e aprovadas.
    • RF11 – O sistema deve calcular e exibir automaticamente quais matérias estão liberadas para a próxima matrícula, com base nos pré-requisitos.
    • RF12 – O sistema deve exibir explicações e ementas básicas de cada matéria da grade.

    Informação e Comunidade

    • RF13 – O sistema deve conter uma Base de Conhecimento (FAQ) com mecanismo de busca para dúvidas frequentes sobre processos universitários (matrícula, editais, etc.).
    • RF14 – O sistema deve permitir que os veteranos cadastrem e respondam dúvidas no fórum da plataforma.
    • RF15 – O sistema deve exibir as datas críticas e os prazos importantes do calendário acadêmico da instituição.

    Engajamento e Gamificação

    • RF16 – O sistema deve atribuir e contabilizar pontos (ou conquistas/badges) para veteranos que auxiliam calouros.
    • RF17 – O sistema deve possuir uma interface para exibir o ranking ou nível de engajamento dos veteranos mais ativos.
